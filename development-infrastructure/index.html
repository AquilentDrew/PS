Healthcare.gov will adapt an Agile (SCRUM) methodology for implementation. This will allow development to be both incremental and iterative, as well as allow active collaboration across healthcare.gov teams.

The development process should be lean, flexible and efficient. This will allow for difficult user-interactions or challenging concepts to be quickly prototyped.

Healthcare.gov should adapt to changes on an iterative basis, based on the evolution of business requirements and users needs. The goal of the system should be to decouple the content, framework, templates, and dynamic application data.  This will allow greater ease in scaling the system and help limit the impact of configuration changes to a specific area of the system.
